# Release Notes — 0.0.0-bootstrap

**Version:** 0.0.0-bootstrap
**Date:** 2026-06-16
**Phase:** Phase 1.5 — SSOT Restructure (this file is the seed for the
release notes workflow; **no features shipped**)

---

## What's New

This is a documentation-only release. No code changed. No crates
renamed. No public API changes. See `cargo test --workspace`
for the live test count at tag time (58 test binaries, 0 failing,
2026-06-16); `cargo build --workspace`, `cargo test --workspace`,
`cargo fmt --all -- --check`, and `cargo clippy --workspace
--all-targets -- -D warnings` are all clean.

The release exists to establish the SSOT (single source of truth)
hierarchy that all future releases will follow:

- **Status SSOT:** [`docs/implementation_tracker.md`](../implementation_tracker.md) (Group → Phase → Step → Packet)
- **Active state pointer:** [`docs/active_context.md`](../active_context.md)
- **Append-only change log:** [`docs/progress_ledger.md`](../progress_ledger.md)
- **Architecture SSOT:** [`docs/system_architecture.md`](../system_architecture.md)
- **Cross-cutting decisions:** [`docs/decisions/`](../decisions/)
- **Rule files:** [`.spiral/rules/`](../.spiral/rules/) (architecture, coding-standards, testing)
- **Role docs:** [`docs/agents/`](../agents/) (8 roles; security, release, onboarding, PROMPT_LIBRARY are new)

## What's Changed

- **Vocabulary:** The time-based `Month` / `Sprint` / `Chunk` / `Item`
  vocabulary is **retired**. The new vocabulary is **Group → Phase →
  Step → Packet**. The `M4.5 Item 8` (Vortex GC rewrite) is now
  `Phase 1 Step 1.6 Packet 1.6.1`.
- **CI:** Three new supply-chain jobs added (`cargo audit`, `cargo
  deny`, `gitleaks`). Not blocking on first run; will flip to blocking
  once green.
- **Path alignment:** `docs/architecture-shared-everything.md` →
  `docs/architecture/design/shared-everything.md`. Three
  `docs/design-*.md` files moved to `docs/architecture/design/`.
  `docs/innovations-backlog.md` → `docs/innovations/backlog.md`.
  `docs/phase1-tasks.md` archived to `docs/archives/phase1-tasks.md`.

## What's Deprecated

- `spiral-html` (Phase 1 Step 1.2, 2026-06-15). Already removed from
  the workspace; this is the formal "deprecated" record.
- `spiral-css` (Phase 1 Step 1.5, 2026-06-16). Forwards to
  `spiral_fmt::css::*`. New code should depend on `spiral-fmt` directly.

## What's Removed

- `html5ever`, `markup5ever`, `tendril` (Phase 1 Step 1.2).
- `cssparser`, `selectors`, `cssparser-macros` (Phase 1 Step 1.5).

## Known Issues

- 23 orphan-export candidates flagged by
  [`scripts/audit-orphan-exports.sh`](../scripts/audit-orphan-exports.sh)
  at the end of the 2026-06-16 implementer loop (down from 34 / 10
  crates at tag time; 11 of the original 34 closed by packets 1.6.1
  / 1.6.3 / 1.6.4).
  All are Phase 1+ skeletons (un-wired by design). Each maps to a
  packet in the implementation tracker. The audit will flip each crate
  from "skeleton" to "OK (all wired)" as the corresponding Step lands.

## Migration Notes

For agents (human or AI) reading this repo:

- **Stop referencing months and sprints.** The vocabulary is Phase N
  → Step N.M → Packet N.M.K.
- **Read `docs/implementation_tracker.md` first.** It is the status
  SSOT.
- **If you find a `Month X` or `M4.x` reference** in any doc, it is
  historical context. The current state is in the tracker.
- **If you are about to write a new file**, the path conventions are
  now:
  - `docs/decisions/NNNN-<slug>.md` — ADR
  - `docs/releases/<version>.md` — Release notes
  - `docs/security/post-mortems/NNNN-<slug>.md` — Incident report
  - `docs/architecture/<subsystem>.md` — Per-subsystem architecture
  - `docs/architecture/design/<slug>.md` — Design-time document
  - `docs/archives/<name>.md` — Archived doc (not SSOT)
  - `docs/agents/<role>.md` — Role contract
  - `docs/agents/PROMPT_LIBRARY.md` — Canonical prompts
  - `.spiral/rules/<topic>.md` — Rule file
  - `crates/spiral-<name>/` — Crate

## Post-bootstrap (2026-06-16, same day)

Four packets shipped after the 0.0.0-bootstrap tag
was cut. They are **not** in the 0.0.0-bootstrap
release notes; the 0.0.0-bootstrap release predates
them by hours. They will appear in the next tagged
release.

- **Packet 1.6.1 — Vortex GC rewrite.** `VortexHeap`
  with per-origin `OriginArena`, `TaggedCell`
  (4-byte header), `GcKey` (versioned + branded),
  stop-the-world mark-sweep. Old `Heap` type
  removed. 22 new tests; GC went 41 → 84 tests.
  See
  [`docs/architecture/design/vortex-heap.md`](../architecture/design/vortex-heap.md)
  §10 for the post-implementation status.
- **Packet 1.6.3 — `spiral_net::Resolver` consumer wiring.**
  The `Resolver` trait is consumed by `spiral-network`
  via `Client<R: Resolver>` (generic bound, not
  `dyn`). Closed 3 orphans in `spiral-net` + 1 in
  `spiral-network`. See
  [`docs/architecture/net.md`](../architecture/net.md)
  §"Filter hook integration" for the call path.
- **Packet 1.6.4 — `spiral-filter` runtime + `FilterHook` in `spiral-core`.**
  The runtime hook (`Filter` struct +
  `match_url::extract_host`) is shipped. The
  `FilterHook` / `Decision` / `Party` types moved
  from `spiral-filter` to `spiral-core` per
  [`docs/decisions/0005-filter-hook-architecture.md`](../decisions/0005-filter-hook-architecture.md)
  to fix a dep-arrow violation. Closed 10 orphans
  in `spiral-filter` + 2 in `spiral-network`. See
  [`docs/architecture/filter.md`](../architecture/filter.md)
  §"Fork 2" for the architecture rationale.
- **Wave A — Doc-drift cleanup (ADR 0005, packets
  1.6.5 unblocked).** 81 doc-drift findings audited
  in
  [`docs/audits/2026-06-16-doc-drift.md`](../audits/2026-06-16-doc-drift.md).
  Wave A (architecture decision) shipped; packet
  1.6.5 unblocked. Waves B–H are doc-only and ship
  in the same implementer loop.
- **Total post-bootstrap test count:** see
  `cargo test --workspace` for the live count
  (58 test binaries, 0 failing, as of 2026-06-16
  end-of-day). 23 orphan candidates across 6
  crates (down from 34 / 10).
