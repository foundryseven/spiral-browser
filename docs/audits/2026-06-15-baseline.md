## 3. Implementation Plan — M4.4 → M4.6 Sprint Window (2c)

Scope: only items in the user-confirmed M4 window. Items marked `✗ auto` proceed on one approval; items marked `⏸ confirm` require per-item approval; item 5 is double-confirmed (security).

### M4.4 — Vendor parsers into `spiral-fmt` (Chunk 3 of first sprint)

**Item 1 — `spiral-fmt` HTML: bring up to green on the 6 panicking `spiral-html` inputs.** ✗ auto.
- **Files:** `crates/spiral-fmt/src/html/tree.rs`, `crates/spiral-fmt/src/html/tokeniser.rs`, `crates/spiral-fmt/tests/e2e.rs`
- **Change:** Extend existing 8 insertion modes to cover inputs that today panic in `spiral-html` (`spiral-html/src/lib.rs:82-318`): nested `<div>`, attribute cases, text merging, malformed auto-close, DOCTYPE, comment in body. Continue using `NodeId` arena from `spiral-dom`. No new deps.
- **Acceptance:** `cargo test --workspace --exclude spiral-html` ≥ 266 passing. The 6 `spiral-html` panic cases reproduce as `spiral-fmt` e2e tests and pass.

**Item 2 — `spiral-fmt` HTML: minimum-viable `<script>` rawtext tokenisation.** ⏸ confirm.
- **Files:** `crates/spiral-fmt/src/html/tokeniser.rs`
- **Change:** Add `ScriptData` and `Rawtext` states to `State` enum (currently 18). Push `<script>` in tree builder; switch to `Rawtext` on `<style>`, `<textarea>`, `<title>`.
- **Acceptance:** E2e tests for `<script>var x = 1;</script>`, `<style>div { color: red; }</style>`, `<textarea>` pass.

**Item 3 — `spiral-fmt` HTML: character reference / entity decoding.** ✗ auto.
- **Files:** `crates/spiral-fmt/src/html/tokeniser.rs`, new `crates/spiral-fmt/src/html/entities.rs`
- **Change:** Decode named (`&amp;`, `&lt;`, `&gt;`, `&quot;`, `&apos;`, `&nbsp;`) and numeric (`&#65;`, `&#x41;`) character references.
- **Acceptance:** Tests pass for `&amp;`, `&lt;`, `&#x4F;`, `&#78;`.

**Item 4 — `spiral-fmt` CSS: from-spec tokeniser + at-rule + selector parser.** ⏸ confirm.
- **Files:** `crates/spiral-fmt/src/css/mod.rs`, new `crates/spiral-fmt/src/css/tokeniser.rs`, `parser.rs`, `selector.rs`
- **Change:** Replace stub with CSS Syntax Level 3 tokeniser. Parse `@media`, `@import`, `@font-face`, `@keyframes`, `@supports`, `@layer`. Selector parsing: type / class / id / universal, attribute selectors, pseudo-classes, pseudo-elements, combinators. Specificity per Selectors Level 4. NO `var()` / `calc()` / cascade in this chunk.
- **Acceptance:** E2e tests for `div.foo#bar[href="x"]:hover > p + ul`, `@media (min-width: 600px)`, `@import url("x.css")`, `@keyframes spin { from { } to { } }`, `@supports (display: grid)`. Replaces `spiral-css::parse_simple` as the canonical implementation.

**Item 5 — `spiral-crypto`: fix the P0 security bug (sha256 + random_bytes).** ⏸ confirm (security — double confirmation required).
- **Files:** `crates/spiral-crypto/src/lib.rs`, `crates/spiral-crypto/Cargo.toml`
- **Change:** Replace `sha256` body with `sha2::Sha256` digest. Replace `random_bytes` with `getrandom::getrandom`. These are the two already-declared but unused upstream deps. **Per-dependency gate required** to flip `sha2` and `getrandom` from `optional = true` to required (or to keep behind a feature flag).
- **Acceptance:** `sha256("abc") == "ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad"`; `random_bytes(32)` differs across two consecutive calls.

**Item 6 — CI: add a workspace exclude for the known-red `spiral-html` tests until `spiral-fmt` replaces it.** ✗ auto.
- **Files:** `.github/workflows/ci.yml`
- **Change:** Add a second test job (or modify the existing one) that runs `cargo test --workspace --exclude spiral-html`. The 6 known panics stay isolated to the migration window.
- **Acceptance:** CI green on all 3 OSes.

**Item 7 — Formalise the agent verification protocol as a `justfile`.** ✗ auto (user confirmed Q5).
- **Files:** new `justfile` at workspace root, `JUSTFILE.md` explaining usage
- **Change:** Recipe chain: `fmt-check → clippy-workspace → test-workspace → smoke-build`. Per Q5 user confirmation.
- **Acceptance:** `just verify` reproduces the steps AGENTS.md mandates, in the right order, with verbatim output.

## 6. Locked decisions (architect recommendations, user-approved)

These are the three open architectural forks that the `architect` subagent reviewed. They are now part of the plan and should not be reopened without explicit instruction.

### Fork 1 — CSS crate boundary after `spiral-fmt` lands → **Option B (re-export shim)**

`spiral-css` becomes a thin re-export shim around `spiral-fmt::css`, with `#[deprecated]` on the shim and deletion scheduled for the sprint after Chunk 3 ships. Mirrors the in-flight `spiral-html` rewire pattern.

- **Option A (delete) rejected:** too aggressive; mirrors no prior pattern in the tree.
- **Option B (shim) chosen:** matches the in-flight `spiral-html` rewire.
- **Option C (deprecation marker, defer deletion) rejected:** accumulates technical debt with no gain.

### Fork 2 — Filter ownership in Bet 1 → **Option B (browser-process-global)**

Single browser-process-global `Filter` instance, with a future `PolicyOverride` parameter designed in for per-context policy. Honours `docs/architecture/design/shared-everything.md:74` which already places the network filter in the per-process shared row.

- **Option A (per-tab) rejected:** duplicates rules and hostnames; doesn't match the Bet 1 shared-everything diagram.
- **Option B (global) chosen:** matches the architecture diagram; future `PolicyOverride` parameter is a non-breaking add.
- **Option C (per-Context) rejected:** more complex than warranted for the M4.5 window; the per-context parameter arrives when `Context<'brand, InProcess>` is threaded through the network call site (M25).

### Fork 3 — Stylesheet wiring in Gyre → **Option A (inline lookup, `ComputedValues` local to Gyre)**

Inline lookup in `spiral-gyre/src/style.rs`; `ComputedValues` is local to Gyre (not in `spiral-core`); `LayoutNode` gains a `style: ComputedValues` field. Future `spiral-style` crate migrates the type without renaming.

- **Option A (inline) chosen:** no new crate boundary in an M4.6 window already at capacity.
- **Option B (ComputedStyle in spiral-core) rejected:** new-crate or new-type design in a window that is already full.
